## Key Design Decisions

### 1. **Fuzzy Matching for Epics**
   - **Why**: Epic names vary (e.g., "User Auth" vs "User Authentication")
   - **How**: difflib.SequenceMatcher with configurable threshold
   - **Default**: 0.75 (balanced between strict and loose)

### 2. **Coverage-Focused (Not Hours)**
   - **Why**: User wants to know WHAT was identified, not time estimation
   - **Exception**: Total hours comparison is separate
   - **Metrics**: Count, percentage, matched/missing/extra

### 3. **Separate Markdown and PDF**
   - **Why**: Markdown is always available, PDF requires dependencies
   - **Fallback**: If PDF fails, Markdown still generated
   - **Option**: `--no-pdf` to skip PDF entirely

### 4. **3-Phase Fuzzy Matching**
   - **Phase 1**: Exact match (fast, common case)
   - **Phase 2**: Fuzzy match (handles variations)
   - **Phase 3**: Identify missing/extra (completeness check)

### 5. **Status Emojis**
   - **Why**: Quick visual assessment
   - **Thresholds**: 80% = âœ…, 50% = âš ï¸, <50% = âŒ
   - **Consistency**: Used throughout report
